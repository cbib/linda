Extract Data files from tabular file!

Using a external tabular file, Users can easily import experimental and environmental data related to their studies. he can also extract experimental design map (field, block, row, column, etc;) and correlate biological material location with this experimental design map.
Sometimes, data are study-specific, like data for a given observation under a given experimental factor condition in a given study. Click to see a example of datafiles to import

Case 1: One column for Study ID

Each row (identified by a given study ID column) of your datafile will be integrated in a new datafile, himself integrated in the corresponding study ID labelled study. You may also want to create study for each id found in your Study ID column if they are not in your investigation tree. Consequently, two options are available:

Link with existing study(ies)

If you want to link your data with to existing studies, your datafile must contain a column dedicated to study ID. Study Ids must be the same as unique Study IDs in your investigation. In this case, you should check Datafiles option !

create study(ies) at flight

If you want to create each study from your data file, your datafile must also contain a column dedicated to study ID. Each Study ID found in your dedicated column will be used to create the basic structure of your MIAPPE study component. This time this column will be used to create study at flight based on their ID (assigned as study Unique ID); Your data file will be splitted accordingly for each associated study ID rows. In this case, you should check Datafiles + studies option !

Case 2: No column for Study Id

In some other cases, data are investigation-specific like global environmental conditions that take place in each study concomitantly. Here, the whole datafile is integrated in each study found in your parent investigation. In this case, you should check No Study ID option !

Data files table

Original headers column

this column contains the headers found in your tabular file. Let's assign the corresponding MIAPPE component to each column. You must have a column for study unique ID !! In this way, the data will be distributed over each study when they are imported.

MIAPPE assignment column

The MIAPPE assignment column allows you to declare a column as MIAPPE component column. It could be a study component, but also an experimental factor, an observed variable, a biological material or a observation units. This way, you inform Linda that this column contains only given MIAPPE component associated data.
